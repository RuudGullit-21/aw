Обработка "АРМ Логиста"

Обработка является основным рабочем местом логиста и предназначена для решения следующих задач:

  • автоматическое (маршрутизация) и ручное формирование документов «Маршрутный лист»;

  • отображение транспортных средств и документов «Заказ на ТС», «Маршрутный лист» на карте;

  • подготовка списка транспортных средств  и документов «Заказ на ТС» для маршрутизации;

  • дополнительное распределение остатков грузов по записанным документам «Маршрутный лист»;

Форма АРМ содержит командную панель формы и панели:

  • дерево заказов 

  • дерево маршрутов

  • статистика

  • планировщик

Командная панель формы содержит:

  • дата для заполнения дерева заказов, дерева маршрутов и планировщика 

  • подменю с кнопками для управления видимостью блоков на форме

  • кнопка переключения расположения элементов на форме

  • кнопка "Карта" для открытия формы обработки "Электронные карты"

  • кнопки "Отбор", "Сортировка", "Настройки" для открытия соответствующих вспомогательных форм АРМ. Иконка на кнопках отображает применение отбора или сортировки. 

  • кнопка "Выписка заказов" для открытия соответствующей обработки для формирования документов "Заказ на ТС" на основании документов "Потребность в перевозке".

Дерево заказов

Содержит список документов «Заказ на ТС», для которых на рабочую дату еще не сформированы маршрутные листы.
Особенности работы с деревом заказов:

  • Возможность фильтрации по зонам, для этого в справочнике "Зоны географического расположения" предусмотрена возможность указания закрепленных логистов и транспортных средств. При проверке попадания пунктов «Заказа на ТС»в зону могут учитываться отдельно пункт отправления, пункт прибытия или оба пункта в зависимости от настроек фильтрации.
  • В дереве возможны различные варианты группировки строк. 
  • Под уровнем заказа отображается уровень грузового и товарного состава. Если заказ имеет детализацию "по заказу в целом" этот уровень не отображается. 
  • Из дерева возможно перетаскивание выделенных флагом документов в дерево маршрутов и планировщик по следующему алгоритму: если текущая строка не выделена флагом - переносится только эта строка, если текущая строка выделена флагом - переносятся все строки, которые отмечены флагом во всем дереве.
  • При выполнении ручной и автоматической маршрутизации флаг в строке дерева снимается и устанавливать в дереве маршрутов напротив созданных документов.

Дерево маршрутов

Содержит список транспортных средств с учетом различных отборов и документов «Маршрутный лист». Предусмотрено отображение как новых документов, полученных в результате маршрутизации, так и уже записанных.
Особенности работы с деревом маршрутов:

  • Возможность гибкой настройки заполнения списка транспортных средств с фильтрацией по различным реквизитам и свойствам, в том числе и по текущему состоянию ТС (для исключения ТС в ремонте), наличию экипажа, принадлежности к зоне.
  • В дереве возможно цветовое оформление различных уровней отдельным цветом фона. Цвет фона указывается в настройках АРМ. 
  • После записи новых документов «Маршрутный лист» новые документы могут быть отправлены на печать. Параметры печати указываются в настройках АРМ.

Планировщик

Предназначен для визуальной оценки доступности и степени загрузки транспортных средств. 
Информация формируется на основании записей регистра сведений "Состояние ТС", из которого выводятся документы "«Маршрутный лист», «Ремонтный лист» и «Диспозиция ТС». 
Командная панель планировщика содержит:

  • переключатель количества дней

  • переключатель детализации в часах

  • кнопка перехода к рабочей дате

  • кнопки перехода на день вперед или на день назад

Планировщик выводится в разрезе двух измерений: время (сверху) и ТС (слева).
На пересечении в виде блоков выводится ячейка в соответствии с состоянием транспортного средства по регистру "Состояние ТС". Длина блока соответствует продолжительности состояния, цвет - из справочника "Состояния ТС". 

На планировщике предусмотрены следующие виды операций:

  • Перетаскивание документов «Заказ на ТС» из дерева заказов для распределения на конкретное транспортное средство. В результате операции в дерево маршрутов и планировщик будет добавлен новый маршрутный лист, время следования между пунктами будет рассчитано по регистру сведений "Расстояние между пунктами", определено с помощью карты, или подставлено по умолчанию из настройки "Время между пунктами по умолчанию". При активизации какого-либо документа в дереве заказов окно отправления и прибытия по заказу на планировщике выделяется особым цветом фона. 

  • Для блоков соответствующих документу «Диспозиция ТС» разрешено изменение длительности блока и изменение транспортного средства, для других документов - только изменение транспортного средства.  Настройка "Спрашивать при изменении даты и транспортного средства на планировщике" позволяет включить или отключить вопрос при выполнении этой операции. 

  • При добавлении записи открывается форма нового документа «Диспозиция ТС». В документ подставляется период, транспортное средство и организация организация ТС, комментарий, по умолчанию подставляется  состояние "заблокирован". После записи нового документа планировщик обновляется автоматически. 

  • При удалении записи задается вопрос об отмене проведения документа-регистратора данного состояния.

  • При нажатии кнопки "редактировать" открывается форма документа-регистратора данного состояния. 

На планировщике запрещены следующие виды операций:

  • Перетаскивание из дерева маршрутов на планировщик.

  • Перетаскивание из планировщика в дерево маршрутов и дерево заказов.

Особенности работы с планировщиком:

  • На планировщике рабочее время организации отображается светло-зеленым цветом фона. Время определяется согласно настройки организации "Время начала работы", "Время окончания работы". Текущая организация определяется по пользователю из настроек АРМ.

  • При активизации строки с транспортным средством в дереве маршрутов это транспортное средство выделяется на планировщике синим цветом и жирным шрифтом.

  • Для документов «Маршрутный лист» в подсказке к блоку выводится детальная информация по весу и объему груза и проценту загрузки. 

Карта

Для использования возможностей картографии предполагается параллельное открытие формы обработки "Электронные карты", которую рекомендуется отображать на отдельном мониторе. Обработка запускается при старте АРМ автоматически. Эта возможность может быть отключена в настройках АРМ.
Обработка "Электронные карты" в режиме работы одновременно с АРМ Логиста дополняет основной функционал АРМ возможностями отображения местоположения транспортных средств, «Заказов на ТС», плановых и фактических маршрутов на карте. 
Используемый тип картографии определяется в форме "Настройки функциональности". 
Особенности работы с картой:

  • Документы «Заказ на ТС», «Маршрутный лист» - новый и записанный, отображаются на карте особым цветом линии, который настраивается в дереве заказов и маршрутов соответственно на уровне документа. Цвет подставляется автоматически из настроек АРМ и может интерактивно изменяться пользователем при необходимости. Отображаются только документы, выделенные в дереве флагом.

  • Документ «Заказ на ТС» отображается на карте в виде прямой соединительной прямой от пункта отправления до пункта прибытия. 

  • Документ «Маршрутный лист» отображение в виде планового маршрута между всеми пунктами, построенного по дорогам механизмом картографии.

  • При активизации строки в дереве заказов или маршрутов соответствующий маршрут отображается более жирным видом линии.

  • Список транспортных средств для вывода на карте совпадает со списком в дереве маршрутов.

  • Режим "указания координат на карте" включается через контекстное меню из дерева заказов и маршрутов для пункта назначения и позволяет указать координаты выбранного пункта на карте. 

Подбор транспортных средств 

Подбор транспортных средств по заказу производится по кнопке в командной панели дерева заказов. По нажатию на кнопку "Подбор" ограничивается список ТС, которые уже находятся в дереве маршрутов, с учетом параметров подбора. При отключении подбора или переходе в списке на другой заказ режим подбора отключается, список ТС возвращается в исходное состояние до наложения подбора. 
При подборе учитываются следующие параметры:

  1. Совпадение требуемого типа ТС с учетом "взаимозаменяемых типов ТС".

  2. Возможность загрузки в ТС с учетом имеющейся загрузки по весу, объему и количеству упаковок.

  3. Доступность ТС для планирования с учетом регистра сведений "Состояние ТС" на дату отправления по заказу.

  4. Доступность ТС по радиусу, указанному в панели разделов (если форма с картой открыта, установлен флаг "при подборе ТС по заказу учитывать радиус" и радиус указан).

  5. Отсутствие просроченных документов на дату отправления по заказу (как в АРМ Диспетчера) 

  6. Возможность своевременного прибытия к точке отправления (ETA) согласно расчетному времени прибытия по плановым данным (плановые даты в маршрутных листах, без учета фактического местоположения).
