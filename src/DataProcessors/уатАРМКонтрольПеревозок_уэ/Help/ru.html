Обработка "АРМ Контроль перевозок"

Предназначен для отслеживания выполнения перевозок по документам. Обработка может работать в режиме отображения документов "Маршрутный лист" или "Путевой лист". Режим определяется в настройках формы.

АРМ позволяет решать следующие задачи:

  • Контроль текущего местоположения транспортных средств с поиском по ключевым полям
  • Контроль отклонения от маршрута
  • Контроль выполнения точек на карте
  • Контроль выполнения заказов с поиском по ключевым полям заказа
  • Контроль предстоящих и прошедших событий в списке
  • Оперативные коммуникации с заказчиком и исполнителем

Интерфейс формы АРМ состоит из панелей:

  • Транспорт
  • Статистика
  • События
  • Карта
  • Пункты назначения

Командная панель формы состоит из следующих элементов:

  • Рабочая дата - для отбора маршрутных документов и событий;
  • Количество дней - для отбора маршрутных документов и событий;
  • Подменю "Панели АРМ" для управления видимостью панелей на форме;
  • Кнопка обновления данных;
  • Кнопка открытия формы настроек АРМ.

Транспорт

Таблица транспортных средств и панель поиска.

Колонки таблицы:

  • Флаг "Отображать на карте".
  • Картинка "Принадлежность" и наименование ТС. 
  • Водитель - определяется по данным текущего маршрутного документа. Если текущий маршрутный документ не определен, отображается "основной водитель", указанный на закладке "Экипаж" в карточке транспортного средства.

На панели поиска расположена строка поиска, кнопки для установки произвольных отборов и флаг "На линии". Поиск выполняется по ключевым полям: наименование ТС, наименование водителя.

Флаг "На линии" позволяет отобрать только те транспортные средства, которые удовлетворяют  следующим критериям:

  • Имеются сведения о местоположении ТС за последние 24 часа от рабочей даты.
  • Имеется маршрутный документ на рабочую дату, на который назначено данное ТС.

На командной панели расположена кнопка "Взаимодействия", по нажатию на которую открывается форма для планирования взаимодействий или непосредственной коммуникации с водителем, выделенным в текущей строке.

Статистика

Содержит информационное поле с дополнительными сведениями по транспортному средству, выделенному в текущей строке:

  • Регистрационный номер и гаражный номер
  • Модель
  • Фактический пробег за весь период по данным спутникового мониторинга
  • Предупреждения по техническому обслуживанию и документам с истекшим сроком действия

События

Содержит таблицу событий, зарегистрированных в системе. Таблица может отображаться в двух режимах в зависимости от переключателя:

  • события по транспортному средству, выделенному в текущей строке (по умолчанию);
  • все события.

Колонки таблицы:

  • Дата
  • Вид события
  • Событие
  • ТС, Водитель. Отображаются в режиме "по всем документам".

Карта

Содержит карту для визуального отображения транспортных средств, а также планового и/или фактического маршрута транспортного средства по текущему маршрутному документу, для проигрывания маршрутов.

Пункты назначения

Содержит пункты назначения по текущим маршрутным документам. Вид документов определяется настройкой: путевые листы или маршрутные листы. Текущие маршрутные документы определяются на рабочую дату по алгоритму:

  • Для невыполненных маршрутных листов документ считается текущим, если плановые интервал выполнения попадает в рабочий интервал.
  • Для выполненных маршрутных листов документ считается текущим, если фактический интервал выполнения попадает в рабочий интервал.
  • Для путевых листов документ считается текущим, если интервал выполнения пунктов задания попадает в рабочий интервал.

Рабочий интервал рассчитывается с учетом настроек "Рабочая дата", "Количество дней". Например, если количество дней указано "3", то рабочий интервал включает рабочую дату и 2 дня до рабочей даты.

Список пунктов может отображаться в двух режимах в зависимости от переключателя:

  • пункты назначения по транспортному средству, выделенному в текущей строке (по умолчанию);
  • пункты назначения по всем текущий маршрутным документам.

Колонки таблицы:

  • Флаг "Выполнен" с возможностью редактирования значения.
  • Картинка "Статус документа" и номер документа.
  • Порядковый номер и наименование пункт назначения.
  • Плановая дата прибытия, убытия
  • Фактическая дата прибытия, убытия с возможностью редактирования значения.
  • ТС, Водитель. Отображаются в режиме "по всем документам".
  • Заказы - содержит номера заказов в данном пункте назначения.
  • Контрагент - содержит список контрагентов по заказам в данном пункте назначения: заказчика, грузоотправителя, грузополучателя (без повторяющихся значений).

На командной панели также расположены следующие элементы:

  • Строка поиска для отбора пунктов назначения по ключевым полям: номер заказа, пункт назначения, контрагент. Если в результате поиска найдены строки - устанавливается отбор на строки. Маршрут по первой из найденных строк отображается на карте.
  • Кнопка "Взаимодействия", по нажатию на которую открывается форма для планирования взаимодействий или непосредственной коммуникации с водителем или контрагентом из Заказов на ТС в текущей строке.
  • Переключатель для управления отображением документов в зависимости от статуса:
    • все
    • только невыполненные
    • только выполненные

  • Кнопка для заполнения фактического посещение пунктов назначения по данным спутникового мониторинга

Особенности работы со списком пунктов:

  • Текст выделяется зеленым цветом:
    • при прибытии в пункт без опоздания
    • при убытии из пункта без опоздания

  • Текст выделяется синим цветом при отклонении фактического времени прибытия от планового и времени отклонения больше допустимого, в случае, если фактическое время посещения пункта в пределах временного окна заказа.

 

  • Текст выделяется красным цветом при отклонении фактического времени прибытия от планового и времени отклонения больше допустимого, в случае, если фактическое время посещения пункта не попадает во временное окно заказа.

     

  • Текст выделяется оранжевым цветом:
    • при опоздании прибытия в пункт. Опоздание рассчитывается для пункта, когда фактическое время прибытия не заполнено и отклонение плановой даты прибытия в пункт от текущего времени больше допустимого
    • при опоздании убытия из пункта. Опоздание рассчитывается для пункта, когда фактическое время убытия не заполнено и отклонение плановой даты убытия из пункта от текущего времени больше допустимого.

Настройки "Допустимое отклонение от времени прибытия" и "Допустимое отклонение от времени убытия" можно установить в «Правах и настройках» . По умолчанию значение данных настроек: 15 минут.

Для управления отображением треков на командной панели АРМ расположена кнопка "Трек". При нажатии на кнопку над картой отображается панель, состоящая из элементов:

  • Флаг "Плановый маршрут" с возможностью проигрывания маршрута.
  • Флаг "Фактический маршрут" с возможностью проигрывания маршрута.
  • Период отображения фактических координат по маршруту. Период заполняется автоматически при выборе какого-либо маршрута в списке пунктов назначения и может быть изменен пользователем.
