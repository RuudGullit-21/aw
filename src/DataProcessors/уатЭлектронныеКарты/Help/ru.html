Обработка "Электронные карты"

Обработка предназначена для просмотра на карте информации о плановых и фактических маршрутах (треках) транспортных средств, определения фактического местоположения транспортных средств и решения других задач, связанных с электронными картами. Обработка поддерживает возможность отображения карт несколько видов: "Яндекс", Open street map. Вид используемых карт указывается в настройках функциональности. 

Для отображения фактических данных используются значения, полученные из системы спутникового мониторинга или из мобильного приложения "Водитель УАТ". Для каждого транспортного средства источник фактических данных задается индивидуально в форме справочника "Транспортные средства".

Обработка может быть открыта из интерфейса программы и использоваться независимо от других объектов или может быть открыта из других объектов конфигурации: документы "Заказ на ТС", "Маршрутный лист", "Путевой лист", справочники "Пункты назначения", "Геозоны", обработка "АРМ Логиста".

Панель маршрутизации позволяет для выбранных пунктов назначения построить плановый маршрут и рассчитать расстояние, время в пути.

Если обработка открывается из АРМ Логиста, на форме обработки становится доступно подменю "Отображение". В подменю настраивается отображение на карте планового маршрута для документов в "Дереве заказов", для незаписанных "Маршрутных листов". Для фактических маршрутных листов доступно отображение фактических маршрутов.

Транспортные средства на карте отображаются в виде иконки.   Расположение иконки соответствует последним координатам транспортного средства.

Возможно два состояния транспортного средства:  

  •  - транспортное средство свободно 
  •  - транспортное средство занято.  Цвет обводки иконки определяется состоянием транспортного средства. Установить цвет состояния можно в справочнике "Состояния ТС".

 Если используется мобильное приложение "Водитель УАТ":

  •  - ТС на линии
  •  - ТС не на линии

Маршруты отображаются на карте в виде линий с указанием направления. Цвет линии определяет вид маршрута:

  • - синий - соответствует выполненному участку планового трека
  • - серый -  соответствует невыполненному участку планового трека
  • - черный - соответствует фактическому треку

При открытии из АРМ Логиста отображаются только плановые треки, а цвет трека настраивается в строке АРМ.

Точки маршрута на карте отображаются в виде иконки. В режиме отображения трека по "Маршрутному листу" или "Путевому листу" внутри иконки отображается порядковый номер точки маршрута, в режиме отображения трека по "Заказу на ТС" вид операции: погрузка, разгрузка. Цвет рамки иконки определяется цветом маршрута. Цвет фона определяется статусом посещения пункта:

  • - белый - пункт не посещен, нет опоздания
  • - оранжевый - пункт не посещен, есть опоздание
  •  - синий - пункт посещен с опозданием, время посещения в пределах временного окна
  • - красный - пункт посещен с опозданием, время посещения за пределами временного окна
  • - зеленый - пункт посещен без опоздания

Опоздание рассчитывается как разница между фактическим и плановым временем прибытия, а если пункт не посещен - как разница между плановым и текущим временем. Рассчитанная разница сравнивается с настройками "Допустимое отклонение от времени прибытия" в "Правах и настройках". Разница больше допустимой считается опозданием. Опоздания для посещенных пунктов рассчитываются только для треков по "Маршрутным листам".

При открытии из справочника "Пункты назначения" в обработке можно указать координаты пункта на карте и просмотреть местоположение пункта на карте.

При открытии из справочника "Геозоны" в обработке можно указать координаты зоны и просмотреть границы зоны на карте.

Команда "Геозоны" предназначена для изменения отображения зон на карте. Данная команда доступна во всех режимах, кроме режима редактирования координат зон.
