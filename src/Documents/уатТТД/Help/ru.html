Документ "Товарно-транспортный документ"

Назначение

Документ «Товарно-транспортный документ» – аналог «бумажных» документов (ТТН и талоны заказчиков), которые отражают работу автомобиля у заказчика и заверяются им. Товарно–транспортные документы вводятся в таблицу ТТД путевого листа.

Табличная часть закладки «Выработка» заполняется автоматически на основании данных закладки «Задание» путевого листа. Также возможно заполнение по кнопкам подменю «Заполнить»

  • При нажатии кнопки «Заполнить по выбранному путевому листу» будут добавлены новые строки выработки по строкам таблицы "Задание" выбранного путевого листа.
  • При нажатии кнопки «Обновить выработку в текущей строке по путевому листу» будет найдена подходящая по аналитикам строка таблицы "Задание" путевого листа из текущей строки, и по ней будет обновлена выработка в текущей строке.
  • При нажатии кнопки «Перезаполнить по путевому листу из текущей строки» будут удалены все строки, содержащие путевой лист из текущей строки, а потом будут добавлены новые строки выработки по строкам таблицы "Задание" этого путевого листа.

    Закладка «Стоимость» содержит расчет стоимости транспортных услуг. Табличные данные заполняются по кнопке «Рассчитать». Рассчитанные данные можно отредактировать или ввести вручную. При расчете используется справочник «Прейскуранты на транспортные услуги».

    Закладка «ТТН» содержит реквизиты, которые используются при печати формы «ТТН».

    Также поддерживается операция "По маршрутному листу и Заказу на ТС". ТТД с такой операцией создается на основании Заказа на ТС или Маршрутного листа. В этом случае ТТД заполняется по кнопке «Заполнить по маршрутному листу и Заказу на ТС».

    Реквизиты

    • Отразить в управленческом/бухгалтерском/налоговом учете. Признак использования документа в определенном виде учета. Используется при работе с типовыми документами.
    • Организация. Ссылается на справочник «Организации». 
    • Контрагент. Ссылается на справочник «Контрагенты». 
    • Договор контрагента. Ссылается на справочник «Договоры контрагентов». 
    • Номер ТТН по учету контрагента. Тип значения: «Строка». Номер документа ТТН контрагента.
    • Номер пачки. Тип значения: «Число». Номер пачки (если ведется учет по пачкам).
    • Ответственный. Ссылается на справочник «Пользователи». Пользователь, ответственный за заполнение документа. 
    • Комментарий. Тип значения: «Строка». Произвольный комментарий. 

    Табличная часть «Выработка»

    • Путевой лист. Ссылается на документ «Путевой лист». Путевой лист, по которому введен данный ТТД. 
    • Маршрут. Ссылается на справочник «Маршруты». Маршрут движения.
    • Груз. Ссылается на справочник «Грузы». Перевозимый груз.
    • Подача. Тип значения: «Дата».  Время подачи, в часах и минутах.
    • Время работы. Тип значения: «Дата».  Время в работе, в часах и минутах.
    • Количество ездок. Тип значения: «Число». Количество ездок. 
    • Количество операций. Тип значения: «Число». Количество операций. 
    • Перевезено за ездку. Тип значения: «Число».  Вес груза, перевезенный за одну ездку, в тоннах.
    • Пробег с грузом за ездку. Тип значения: «Число». Пробег за одну ездку с грузом, в км.
    • Пробег порожний за ездку. Тип значения: «Число». Пробег за одну ездку порожний, в км.
    • Погрузочно-разгрузочные работы. Тип значения: «Число». Время, вес или работа по погрузке/разгрузке, в часах, тонно-часах или тоннах.
    • Простой под погрузкой. Тип значения: «Число». Время простоя при погрузке, в часах и минутах. 
    • Простой под разгрузкой. Тип значения: «Число». Время простоя при разгрузке, в часах и минутах. 
    • Пробег специальной работы. Тип значения: «Число». Пробег специальной работы (может быть введено до 5 различных видов спец. работ), в км.
    • Выработка, Параметр выработки. Здесь вводится произвольный параметр выработки (в справочнике Параметры выработки можно создавать дополнительные пользовательские параметры выработки) и его значение.
    • Прицеп/оборудование 1, Прицеп/оборудование 2. При заполнении ТТД по Путевому листу сюда переносятся значения прицепов и оборудования из Задания, которые используются при расчете стоимости услуг для отбора тарифов по областям действия "Прицеп", "Модель прицепа", "Тип прицепа".

    Табличная часть «Стоимость»

    • Путевой лист. Ссылается на документ «Путевой лист». Путевой лист, по которому введен данный ТТД.
    • Маршрут. Ссылается на справочник «Маршруты». Маршрут движения.
    • Объект строительства. Ссылается на справочник «Объекты строительства». Объект строительства, по которому выполняется задание.
    • Вид перевозки. Ссылается на справочник «Виды перевозок». Вид транспортной услуги.
    • Груз. Ссылается на справочник «Номенклатура». Перевозимый груз.
    • Параметр выработки. Ссылается на справочник «Параметры выработки ТС и водителей». 
    • Тариф. Ссылается на справочник «Тарифы на транспортные услуги». Тариф, по которому рассчитана сумма в строке.
    • Количество параметра выработки. Тип значения: «Число». Количество по параметру выработки.
    • Услуга. Ссылается на справочник «Номенклатура». Номенклатура услуги по перевозке груза. 
    • Количество. Тип значения: «Число». Количество транспортных услуг. 
    • Цена. Тип значения: «Число». Цена транспортных услуг.
    • Сумма. Тип значения: «Число». Сумма транспортных услуг. 

    Закладка «ТТН»

    На закладке находятся дополнительные реквизиты и реквизиты, предназначенные для вывода на печать товарно-транспортной накладной:

    • Грузоотправитель. Ссылается на справочник «Контрагенты». Контрагент, отправляющий груз.
    • Грузополучатель. Ссылается на справочник «Контрагенты». Контрагент, получающий груз.
    • Пункт погрузки. Тип значения: «Строка», ссылается на справочник «Пункты назначения». Адрес отправления груза.
    • Пункт разгрузки. Тип значения: «Строка», ссылается на справочник «Пункты назначения». Адрес доставки груза.
    • Переадресовка. Тип значения: «Строка», ссылается на справочник «Пункты назначения». Адрес переадресовки груза. Адрес пункта, куда необходимо перенаправить груз в момент его доставки в пункт назначения.
    • Вид перевозки. Ссылается на справочник «Виды перевозок». Вид перевозок.
    • Табличная часть "ТТН". Это расширение табличной части "Выработка". Поля используются для вывода на печать ТТН.

    Печатная форма «Заказ-наряд»

    По документу «Товарно-транспортный документ» возможна печать «Заказ-нарядов». Данные для печатной формы будут получены из документов «Путевой лист», указанных в табличной части «Стоимость». Для каждой различной комбинации путевого листа и маршрута будет сформировано по одной печатной форме. Для заполнения реквизитов «Тариф» печатной формы будут использованы строки с параметрами выработки «Пробег общий» и «Время в работе». Сумма документа будет получена только по вышеуказанным строкам.

    Особенность заполнения грузового состава ТТД на вкладках ТН и ТТН в ручном режиме и при вводе на основании

    Если документ ТТД вводится на основании Заказа на ТС и Маршрутного листа или в нем установлена одноименная операция и выполняется ручное заполнение таблицы грузов, то предусмотрен следующий алгоритм заполнения видов упаковок и единиц измерения:
    • При добавлении в таблицу номенклатуры груза заполняется поле "Единица измерения";
    • При добавлении в таблицу грузового места, не являющегося контейнером заполняется поле "Вид упаковки";
    • При добавлении в таблицу грузового места, являющегося контейнером заполняется поле "Единица измерения" значением типа контейнера, в поле "Вид упаковки" подставляется предопределенное значение "Контейнер".

    Особенность заполнения ТТД по FTL «Заказам на ТС»

    Если документ ТТД создается на основании Заказа на ТС с установленным признаком FTL и Маршрутного листа, то будут созданы отдельные ТТД на разные грузы. В этом случае несколько грузов, у которых совпадают пункты погрузки и пункты разгрузки, будут объединены в один ТТД, а грузы с несовпадающими пунктами погрузки и разгрузки попадут в разные ТТД.

    Если ТТД заполняется по кнопке «Заполнить по маршрутному листу и Заказу на ТС», то 

    • Если пункты погрузки и разгрузки в ТТД не заполнены, то поведение будет таким, как описано выше, с возможным созданием новых ТТД.
    • Если пункты погрузки и разгрузки в ТТД заполнены, то из Заказа на ТС будут выбраны только те грузы, для которых именно эти пункты являются пунктами погрузки и разгрузки.

    Особенности расчета выработки по этапам

    При расчете выработки по ТТД в Путевом листе или услуг в ТТД происходит свертка строк табличной части "Выработка", для которых совпадает путевой лист и номер этапа (кроме тех строк, у которых номер этапа равен 0). При этом все строки с данными номером этапа и путевым листом заменяются на одну виртуальную строку, в которой ссылочные аналитики (маршрут, объект строительства и др.) будут взяты из первой такой строки, а в качестве значений параметров выработки (пробег, время в работе, количество спецработ и др.) будут взяты сумма или максимум по всем таким строкам, в зависимости от настроек данного параметра выработки. При этом заданный вручную параметр выработки в колонке "Параметр выработки" будет обрабатываться как еще одна спецработа.
