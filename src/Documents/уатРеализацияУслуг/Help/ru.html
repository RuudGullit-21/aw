Документ "Реализация услуг"

Документ предназначен для отражения факта оказания услуг.

Относится к подсистеме "Сервис".

Реквизиты шапки документа

  • Свертка услуг для отражения в регл. учете. Если флаг установлен, то становится видимой табличная часть "Услуги (регл.)", данные которой используются в печатных формах документа и при выгрузке в типовые конфигурации;
  • Организация. Организация, от имени которой оформляется документ;
  • Контрагент. Получатель услуг;
  • Договор контрагента. Договор, по которому оказываются услуги;
  • Ответственный. Пользователь, ответственный за документ.
  • Счет на оплату (сделка). Документ Счет на оплату покупателю, по которому выполнена реализация. Используется при ведении взаиморасчетов "по счетам". 
  • Сметная стоимость. Используется для заполнения в печатной форме "Акт КС-2".
  • Накладные расходы. Используется для заполнения в печатной форме "Акт КС-2".
  • Плановые накопления. Используется для заполнения в печатной форме "Акт КС-2".
  • Дата платежа (план). Дата оплаты по Заказам на ТС по платежному календарю. Если установлен флаг "Указать вручную", то используется указанное значение, иначе дата платежа будет рассчитана автоматически по графику оплаты в договоре контрагента.

Заполнение табличной части "Услуги"

Табличная часть может быть по желанию пользователя заполнена автоматически за произвольно выбранный период времени или еще не выставленными услугами по кнопке "Заполнить". В табличную часть попадают данные  из регистра накопления накапливаемые при проведении «Товарно-транспортных документов», относящихся к данному заказчику. Детальность заполнения табличной части (сумма услуг по объекту в целом, либо количество услуг по каждому параметру выработки, либо по каждой модели транспортных средств, либо по каждому транспортному средству в отдельности) определяется настройкой Учетной политики Управления автотранспортом (закладка Прейскуранты, реквизит Детализация выработки для клиента).

Заполнение табличной части "Услуги (регл.)"

Табличная часть предназначена для отражения услуг в регламентированном учете, если используется свертка (установлен флаг "Свертка услуг для отражения в регл. учете"). При этом данные используются при выводе документа на печать и обменах с типовыми конфигурациями 1С. Не редактируется. Заполняется по данным табличной части "Услуги (упр.)" при установке флага "Свертка услуг для отражения в регл. учете" автоматически или может быть заполнена вручную при нажатии на кнопку "Заполнить / Обновить по данным услуг (упр.)".

Платежный календарь

1) Документ увеличивает задолженность по Платежному календарю, если включен функционал и установлено ведение взаиморасчетов по договору "По накладным", "По счетам", "По заказам на ТС".
Если в договоре указан сложный график оплаты (2 и более этапа), то вся задолженность распределяется по процентам оплаты каждого этапа. Общая сумма задолженности берется по данным регистра Предоставленные услуги с отбором по конкретным Заказам на ТС, таким образом учитываются не только услуги, рассчитанные в самих заказах, но и прочие связанные услуги по заказам, указанные в других документах, например, в Маршрутном листе. При распределении задолженности учитываются уже введенные ранее документы, то есть уже распределенная ранее задолженность. 
2) При проведении этого и других документов, связанных с Заказом на ТС, выполняется расчет Дат платежей по Платежному календарю при условии наступления событий согласно графика оплаты по договору. Даты платежей привязывают этапы оплаты к конкретным датам, это удобно видеть в отчете Платежный календарь.
Если дата платежа указана вручную, то график оплаты по договору игнорируется, вся задолженность по документу распределяется на один этап оплаты "Произвольная дата".

Способ вызова

Вызывается из меню "Работа ТС --> Услуги --> Реализация услуг"
