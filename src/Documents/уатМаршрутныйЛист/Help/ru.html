Документ "Маршрутный лист"

Документ предназначен для формирования и регистрации маршрута транспортного средства по развозке грузов. Формирование маршрута может осуществляться на основании введенных ранее "Заказов на ТС" (операция "Заполнение маршрута по заказам на ТС") или формирования произвольного списка развозки грузов (операция "Ручное заполнение табличной части").  

Документ позволяет оформить перевозку собственным или привлеченным транспортным средством. В случае перевозки собственными транспортным средством перевозчик выбирается из справочника "Организации", водители - из справочника "Сотрудники". В случае перевозки привлеченными транспортным средством перевозчик выбирается из справочника "Контрагенты", водители - физические лица по регистру сведений "Сотрудники контрагентов".

Документ выполняет закрытие "Заказов на ТС" в несколько этапов. Проведенный документ закрывает остатки по регистру "Грузы к перевозке" для того, чтобы груз не был повторно добавлен в другие "Маршрутные листы". При установке флага "Выполнен" проведенный документ закрывает остатки по регистру "Грузы в пути", "Заказы грузоотправителей", что означает факт успешной перевозки груза.

При необходимости закрытия заказа, если перевозка не была выполнена фактически, вводится документ "Закрытие заказов на ТС".

Если в заказе используется вид доставки с собственным складом, то при погрузке/разгрузке выполняется контроль ввода актов передачи на склад комплектации / поступления на склад комплектации соответственно (только для пунктов с установленным флагом "Выполнено").

Реквизит "Есть расхождения в актах" отражает наличие расхождений между плановыми и фактическими данными по связанным с маршрутным листом Актами приемки и отгрузки. Флаг может устанавливаться в значение "Истина" автоматически при проведении актов с расхождениями. В списке документов установленный флаг отображается картинкой

Статусы.

Документ может принимать один из следующих статусов, которые определяются по флагу "Выполнен" в дереве или таблице Маршрут:

  • "Новый" - документ введен, ни один груз не отправлен,
  • "Отправлен" - отправлен хотя бы один груз, но ни один не доставлен,
  • "Частично выполнен" - доставлен хотя бы один груз, но не все,
  • "Выполнен" - доставлены все грузы,
  • "Рассчитан" - доставлены все грузы и выполнен расчет итогов. 

Маршрут.

В документе представляется маршрут транспортного средства с детализацией по грузам - номенклатуре или грузовым местам.

При формировании Маршрутного листа с видом операции "Заполнение маршрута по Заказам на ТС" в табличную часть можно добавить только неразвезенные грузы по ранее введенным заказам и произвольные пункты назначения. Маршрут представляется в виде дерева, в котором отображаются следующие параметры: 

  • время и расстояние до следующего пункта маршрута;
  • планируемое время выполнения операция по заказу и суммарное время стоянки в пункте;
  • планируемое и фактическое времена прибытия/убытия из пункта;
  • временное окно доступности пункта;
  • операция выполняемая в данном пункте по заказу;
  • параметры грузов.
При формировании Маршрутного листа с видом операции "Ручное заполнение табличной части" в маршрут могут быть добавлены произвольные пункты и грузы. Контроль остатков при проведении в данном случае не выполняется. Для отображения пунктов назначения в задании Путевого листа, введенного на основании Маршрутного листа, необходимо заполнить колонку "Тип точки маршрута". 

В подменю "Заполнить" доступны следующие операции:

  • "По заказу на ТС" - заполнение неразвезенных грузов по выбранному заказу;
  • "По остаткам" - заполнение неразвезенных грузов по всем невыполненным заказам;
  • "Плановое время из Заказа на ТС - заполнение планового времени исполнения маршрута в соответствии с Заказом на ТС;
  • "Фактические показатели по плановым" - заполнение фактического времени, расстояния, стоянок, ожидания, времени прибытия и убытия по плановому, установка флага выполнения в каждом пункте маршрута; 
  • "Плановое время и расстояние по карте" - алгоритм заполнения времени и расстояния приведен в справке к справочнику "Маршруты";
  • "Обновить ТЧ "Маршрут" по Заказу на ТС" - обновление пунктов, номенклатуры и весо-объемных характеристик в соответствии с Заказом на ТС из текущей строки
  • "Обновить Маршрутный лист по Заказу на ТС" - обновление пунктов, номенклатуры и весо-объемных характеристик в соответствии с Заказом на ТС из текущей строки, а также обновление реквизитов шапки Маршрутного листа по Заказу на ТС (организации, подразделения, направления перевозки, маршрута, вида перевозки и объекта строительства)
  • Очистить - удаление всех строк.

Пользователь может менять порядок следования пунктов. Перетаскивание, в результате которого пункт разгрузки окажется выше соответствующего ему пункта погрузки невозможно. При проведении документа производится контроль остатков по необработанным "Заказам на ТС".

В дереве маршрутов имеется флаг "Рекламация". При вводе документа "Рекламация" в него попадают только строки "Маршрутного листа", отмеченные данным флагом. В дереве выводится динамическая колонка, которая содержит список введенных "Рекламаций".

Выработка.

Содержит выработку ТС и сотрудников. Выработка одинакова для ТС и сотрудников. Выработка рассчитывается при нажатии на кнопку "Расчет итогов".
  • Сводно. Выработка в свернутом по параметру выработки виде, без учета значений аналитик и Заказов. При создании документа заполняется по умолчанию всеми доступными параметрами выработки. Выработка для "построчных" параметров не редактируется. Можно вручную изменять выработку для пользовательских параметров выработки.
  • Подробно. Выработка в полном виде по Заказам и в целом по документу, содержит аналитики. При создании документа таблица пустая и заполняется при расчете итогов по документу. Подробная выработка полностью доступна для редактирования, после расчета итогов можно скорректировать то или иное значение. 
При изменении или добавлении выработки вручную устанавливается флаг "Ручной ввод". Такие строки выработки при перерасчете итогов не перезаполняются.

Доходы и расходы.

Дополнительные оказываемые услуги вводятся в соответствующей табличной части. Имеется возможность автозаполнения табличной части по тарифам, по шаблонам, распределения по заказам. 

При включенной настройке программы "Учет план-факта в Маршрутных листах" доходы рассчитываются по фактической выработке, а расходы - плановые по плановой выработке, фактические по фактической.

В случае использования операции "Ручное заполнение табличной части" есть ряд особенностей, связанных с проверкой областей действия тарифов. Для некоторых видов областей действия список значений параметров формируется по строкам табличной части "Маршрут". При проверке области действия условия проверяются для этого списка целиком.

  • Для параметров областей действия "Номенклатура" и "Контрагент" используются все строки табличной части. 
  • Для параметров областей действия "Пункт отправления" и "Геозона отправления" используются только строки табличной части с типом точки "Погрузка" или "Дополнительная погрузка". 
  • Для параметров областей действия "Пункт назначения" и "Геозона назначения" используются только строки табличной части с типом точки "Разгрузка" или "Дополнительная разгрузка".
Например, для маршрута "Москва" (погрузка) -> "Тверь" (разгрузка) -> "Санкт-Петербург" (дополнительная разгрузка) в список пунктов назначения попадут "Тверь" и "Санкт-Петербург". Поэтому условия «Пункт назначения в списке "Москва, Тверь, Санкт-Петербург"» и «Пункт назначения не равно "Санкт-Петербург"» будут выполнены, а условие «Пункт назначения равно "Тверь"» - не будет.

Распределение доходов и расходов выполняется:

  • автоматически при проведении закрытого документа. При этом изменяются табличные части документа. При распределении по заказам ТС выбираются все заказы из маршрутного листа, и рассчитывается коэффициент распределения для каждого заказа согласно настроек в статье доходов ил расходов. Затем каждая строка услуги разбивается согласно рассчитанным коэффициентам на несколько строк и в табличной части заполняется колонка "Заказ на ТС";
  • с помощью формы "Распределение услуг по заказам и ТС". Алгоритм распределения аналогичен автоматическому, но пользователь может редактировать состав услуг и выбрать способ распределения иной, чем в настройках.
  • вручную путем указания заказа в табличной части.

При проведении по регистрам "Оперативные доходы" и "Оперативные расходы" в измерения Объект строительства, Маршрут, Вид перевозки, Направление перевозки пишутся значения реквизитов документов "Заказ на ТС" (на которые распределились доходы/расходы), а в измерение Подразделение - подразделение маршрутного листа.

Сопроводительные документы.

На закладке указывается перечень сопроводительных документов по Маршрутному листу и текущий статус обработки документов.

В подменю "Заполнение" доступны следующие операции:

  • Заполнить по шаблону. Открываются формы сопроводительных документов, по одному на каждый вид документа шаблона, соответствующие реквизиты заполняются из шаблона. После заполнения остальных реквизитов и записи справочника, он появляется в таблице.
  • Установить статус. Открывается форма нового документа "Установка статусов сопроводительных документов", табличная часть заполняется выделенными сопроводительными документами. После указания статусов, заполнения других реквизитов и проведении документа статус сопроводительных документов изменяется.
  • Установить срок обработки. Для всех выделенных сопроводительных документов указанным значением заполняется реквизит "Дата обработки", устанавливается вариант заполнения "Вручную". При этом дата обработки будет отражена в учете только при проведении Маршрутного листа.
  • Принять документы заказчика, перевозчика, по пункту. Открывается форма нового документа "Установка статусов сопроводительных документов", табличная часть заполняется всеми сопроводительными документами заказчика, перевозчика или отобранными по пункту назначения соответственно. Проставляется статус "Обработка завершена". После проведения документа сопроводительные документы становятся принятыми (обработка завершена).
  • Принять все документы. Открывается форма нового документа "Установка статусов сопроводительных документов", табличная часть заполняется всеми сопроводительными документами Маршрутного листа. Проставляется статус "Обработка завершена". После проведения документа сопроводительные документы становятся принятыми (обработка завершена).
  • Отменить принятие всех документов. Для всех сопроводительных документов с текущим статусом "Обработка завершена" выполняется отмена установки этого статуса: либо отменяется проведение документа "Установка статусов сопроводительных документов", либо удаляется строка из этого документа, если в нём содержится несколько сопроводительных документов. Если статус "Обработка завершена" был установлен документом "Пакет сопроводительных документов", то выдается сообщение о невозможности отмены статуса по причине запрета программного изменения Пакета.
  • Удалить. Выделенные сопроводительные документы помечаются на удаление, история статусов очищается.

По кнопке "Уведомить перевозчика" создается уведомление по правилу со значениями реквизитов: тип объекта = "Маршрутный лист", тип события "Уведомление перевозчику по сопроводительным документам". В уведомление попадает список необработанных документов перевозчика. Шаблон сообщения в правиле уведомления может быть создан с помощью команды заполнения "Создать шаблон "Уведомление перевозчику"". Отправка сообщения адресату выполняется с помощью регламентных заданий "Рассылка уведомлений" и "Получение и отправка электронных писем" (для почты), при этом присутствует некоторая задержка связанная с расписанием регламентных заданий.

В списке документов "Маршрутный лист" в виде одной из картинок отображается статус обработки сопроводительных документов 

Зарплата.

Содержит данные по расчету зарплаты водителям и прочим сотрудникам. Возможен расчет зарплаты как собственным, так и привлеченным сотрудникам, в зависимости от принадлежности ТС. 
Верхняя таблица содержит перечень сотрудников документа с суммами зарплаты, нижняя расшифровку по видам начислений.
Зарплата заполняется при расчете итогов по документу и использует данные выработки и тарифов зарплаты, которые устанавливаются документом Установка тарифов зарплаты.

Планировщик.

Кнопка "Планировщик" предназначена для включения режима отображения планировщика. Планировщик предназначен для визуального планирования этапов маршрутного листа. Этап маршрутного листа отображается в виде блока соответствующего цвета: 

  • пункт погрузки - желтый; 
  • пункт разгрузки - розовый; 
  • промежуточный пункт - оранжевый; 
  • транзитное время следования между пунктами - светло-зеленый; 
  • ожидание посещения пункта - светло-синий.
Допускается возможность изменения времени прибытия в пункт и убытия из пункта путем перемещения блоков на планировщике.

Дополнительно.

На закладке по кнопке "Рассчитать" можно рассчитать нормативный расход ГСМ. Кнопка "Расшифровка" отображает список видов норм.

Закрытие документа.

При закрытии документ блокируется от изменений, проводки по регистрам выполняются на дату закрытия.
Закрытие документа выполняется автоматически при расчете итогов или вручную по кнопке "Закрытие" верхней командной панели, или простой установкой реквизита "Дата закрытия" на закладке "Дополнительно" и последующим проведением документа.
