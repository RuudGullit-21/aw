Бизнес-процесс "Согласование"

Назначение

Этот бизнес-процесс обходит всех участников согласования (рецензентов) по заранее составленному списку, после чего возвращается к автору (инициатору бизнес-процесса). В случае отрицательного результата согласования автор имеет возможность повторить согласование. В случае положительного результата документ, который является предметом бизнес-процесса, можно провести. Список документов визирования (согласования) см. ниже (доп. информация).

При заполнении списка рецензентов можно указывать исполнителей (пользователей). Предусмотрено два варианта обхода согласующих - "все вместе" и "по очереди". Если установлен тип согласования "по очереди", то задача следующему рецензенту формируется только после завершения задачи предыдущему.

Этот бизнес-процесс можно создавать непосредственно из документов, при помощи меню "Создать на основании" или нажав на гиперссылку Не согласован, которая появляется на форме, только если требуется согласование документа.

Закладка "Все задачи" предназначена для просмотра текущего состояния бизнес-процесса (списка выполненных и не выполненных задач).

Если бизнес-процесс стартован, то изменить его данные невозможно, необходимо создать новый бизнес процесс, а текущий пометить на удаление.

Относится к подсистеме: «Управление персоналом».

Способ вызова

Вызывается из раздела "Администрирование" -> "Визирование документов (уат) -> Согласование"

Дополнительная информация

Вводится на основании документов Ввод ТС в эксплуатацию, Перемещение ТС, Выбытие ТС из эксплуатации, Заявка на расходование денежных средств.

Для работы с данным бизнес-процессом необходимо включить использование типового механизма "Бизнес-процессы и задачи" в форме "Органайзер".

Механизм согласования документов

Для некоторых видов документов можно использовать данный механизм согласования. Для этого в настройках программы устанавливается флаг "Использовать согласование документов" (см. Дополнительная информация).
Для документов какого-либо вида можно включить автоматическую установку флага "Требует согласования" при создании документа в настройках по организации (Установка прав и настроек/Документ требует согласования).
А также запретить каким-либо пользователям изменять значение флага в настройках по пользователям (Установка прав и настроек/Разрешить изменять параметры согласования).
Если флаг "Требует согласования" установлен, но документ не согласован, то запрещено его проведение. После согласования документа проведение разрешается.

Особенность согласования документа "Ремонтный лист"

Для документа "Ремонтный лист" допускается проведение несогласованного документа. В этом случае будут выполнены движения документа только по регистру "Состояния ТС", аналогично случаю, когда в документе "Ремонтный лист" не установлен флаг "Выполнен". После согласования документ автоматически будет перепроведен, и будут выполнены движения по всем регистрам.
