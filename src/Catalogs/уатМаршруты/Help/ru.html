Справочник "Маршруты"

Назначение

Справочник "Маршруты" предназначен для хранения информации об основных маршрутах движения автомобилей. Данные этого справочника используются при заполнении таких документов, как  "Заказ на ТС", "Потребность в перевозке", "Путевой лист", а так же в справочниках "Прейскуранты на транспортные услуги" и "Тарифы зарплаты".

Относится к подсистеме "Транспорт".

Реквизиты

  • Статус. Возможные значения: "действует", "не действует". Недействующие маршруты не отображаются в списке выбора и при подборе по строке в документах.
  • Номер маршрута. Дополнительный номер или код маршрута.
  • Расстояние. Общее расстояние между всеми пунктами назначения, км.
  • Время в пути. Общее время в пути в формате ч:мин.
  • Условие работы. Используется для заполнения условия работы при выборе маршрута в строке табличной части "Задание" документа "Путевой лист".
  • Использовать в АРМ Логиста. Только в версии КОРП. Настройка доступна в режиме представления "Одиночные пункты".
  • Ночной маршрут. Только в версии КОРП. Настройка доступна в режиме представления "Одиночные пункты" при включенном флаге "Использовать в АРМ Логиста". Если флаг установлен, при маршрутизации в АРМ Логиста по алгоритму "Фиксированные маршруты" дата выезда по маршрутному листу будет назначена на 1 день меньше рабочей даты. Флаг предназначен для фиксированных маршрутов, для которых дата завершения маршрута - следующий день после даты отправления.
  • Парные, одиночные (представление маршрута). Список этапов маршрута может быть представлен как в виде одиночных пунктов (что, например, удобно для пассажирских перевозок) так и в виде пар пунктов следования. Эта информация используется при заполнении расписаний движения по маршрутам.
  • Цвет на карте. Цвет для отображения маршрута на карте. Данный реквизит доступен для редактирования, если в (Администрировании - Администрировании УАТ - Настройках программы) включена настройка "Использовать электронные карты". 

Табличная часть «Пункты следования»

Табличная часть заполняется промежуточными пунктами следования по маршруту.

Команда "Оптимизация" доступна только в версии КОРП, и только для представления маршрута одиночными пунктами. Эта команда позволяет взять маршрут, в котором первая точка - погрузка, а все остальные - разгрузки, и оптимизировать его по расстоянию, то есть изменить порядок точек разгрузки так, чтобы минимизировать длину маршрута. Алгоритм маршрутизации зависит от настроек пользователя в АРМ Логиста. Если настроено использование маршрутизации Яндекс, то для перестроения маршрута используется Яндекс маршрутизация, иначе - упрощенный вариант алгоритма Очередей.

  • Грузополучатель. В первой строке маршрута в этом поле указывается грузоотправитель, в остальных - грузополучатель.
    Поле используется для выбора маршрутов в АРМ Логиста при маршрутизации алгоритмом фиксированных маршрутов, если в Заказе на ТС не указан маршрут явно. Если грузополучатели не заполнены, то маршрут может использоваться для всех грузополучателей.

Табличная часть «Расписание рейсов»

В табличной части отображается расписание всех рейсов по маршруту. 

Редактирование расписания на данной форме невозможно. Команда "Расписание" открывает обработку Расписание рейсов, предназначенную для редактирования расписания.
