Справочник "Договоры контрагентов"

Назначение

Справочник предназначен для хранения договоров взаиморасчетов, заключенных с контрагентами. Подчинен справочнику «Контрагенты».

Относится к подсистеме "НСИ".

Реквизиты

  • Организация. Ссылается на справочник «Организации». Организация договора.
  • Контрагент. Ссылается на справочник «Контрагенты». Контрагент, которому принадлежит договор.
  • Входит в группу. Группа договора.
  • Наименование, код. Наименование, код договора.
  • Вид договора. Вид договора с контрагентом. Может принимать следующие значения:
    • С поставщиком
    • С покупателем
    • С комитентом
    • С комиссионером 
    • Прочее - по договорам с видом "Прочее" не ведутся взаиморасчеты по сделкам.
  • Номер и дата. Номер и дата создания договора.
  • Ведение взаиморасчетов. Способ ведения взаиморасчетов с контрагентов. Может принимать следующие значения:
    • По договору в целом - способ ведения взаиморасчетов не требует обязательного указания сделки в документах, сделка может быть указана в качестве дополнительной информации, но задолженность будет отражаться по договору без детализации по сделкам;
    • По счетам - способ ведения взаиморасчетов предполагает обязательное указание счета на оплату (покупателя, поставщика) в документах;
    • По накладным - способ ведения взаиморасчетов предполагает обязательное указание накладной (поступление, реализация) в документах;
    • По заказам - способ ведения взаиморасчетов предполагает обязательное указание заказа на ТС в документах.
  • Валюта взаиморасчетов. Ссылается на справочник «Валюты». Валюта взаиморасчетов по договору. Значение не может быть изменено, если по договору проведен хотя бы один документ.
  • Расчеты в условных единицах. Тип значения: «Булево». В УАТ используется как информационный реквизит, и для синхронизации данных с типовыми конфигурациями, где он определяет возможность ведения расчетов в условных единицах для договоров вида «С покупателем» или «С поставщиком».
  • Вид взаиморасчетов. Ссылается на справочник «Виды взаиморасчетов». Аналитический признак для дополнительного разделения взаиморасчетов.
  • Тип цен. Ссылается на справочник «Типы цен номенклатуры». Тип цен, используемый в объединенных (УАТ + типовая) конфигурациях при формировании типовых документов.
  • Контролировать сумму задолженности. Флаг определяет, будет ли контролироваться общая сумма дебиторской задолженности по договору в соответствии с указанной суммой допустимой задолженности по договору.
  • Допустимая сумма задолженности. Допустимая сумма дебиторской задолженности, при превышении которой в документах «Заказ грузоотправителя», «ТТД» будет выдано информационное сообщение о превышении максимальной суммы задолженности.
  • Расчеты в условных единицах. Используется для синхронизации с типовыми конфигурациями.

Закладка "График оплаты"

Доступна при включенной настройке программы "Платежный календарь" и при ведении взаиморасчетов "По накладным", "По счетам", "По заказам на ТС".
Содержит график оплаты по Заказам на ТС по платежному календарю. Можно либо не использовать график, в этом случае задолженность не возникает и не отображается в платежном календаре, либо использовать один или несколько этапов оплаты. 
При выборе Этапа оплаты из него автоматически подставляются значения в договоре. Указание самого этапа необязательно, он используется для отображения в отчетах платежного календаря.
Вариант отсчета определяет событие, относительно которого будет рассчитана плановая дата платежа:
1) До отправления по заказу - задолженность возникает до отправки груза Маршрутным листом на дату отправления по Заказу. Если выбран этап с этим вариантом отсчета и не внесен платеж по этому этапу, то проведение Маршрутного листа с таким Заказом на ТС будет запрещено.
2) После отправления по заказу, до прибытия по заказу, после прибытия по заказу - задолженность возникает после отправки груза на дату отправления по Заказу, до и после прибытия груза на дату прибытия по Заказу соответственно.
3) После доставки груза - задолженность возникает при доставке груза (выполнение Маршрутного листа);
4) После получения сопроводительных документов - задолженность возникает при получении сопроводительных документов нашей компанией или заказчиком (установка статуса сопроводительного документа "Обработка завершена"), отбор по виду документа задается в реквизите "Вид документа" (совпадает с видом документа в "Заказе на ТС" на закладке "Документы").
Срок - количество дней до или после события, по которому рассчитывается плановая дата платежа.

Закладка "Виды сопроводительных документов"

Предназначена для настройки автоматического расчета плановой даты обработки сопроводительных документов (описание см. в справочниках Сопроводительные документы и Виды сопроводительных документов).
