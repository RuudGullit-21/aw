Справочник "Сопроводительные документы"

Назначение

Данный справочник содержит список сопроводительных документов (СД) на перевозку. 

Относится к подсистеме "Логистика".

Реквизиты

  • Вид документа. Ссылается на справочник «Виды сопроводительных документов». Вид СД.
  • Номер документа. Тип значения: «Строка». Номер СД.
  • Дата документа. Тип значения: «Дата». Дата создания СД.
  • Шаблон. Признак использования в качестве шаблона для создания экземпляров СД по шаблонам перевозок. Только в версии КОРП.
  • Количество экземпляров. Тип значения: «Число». Количество экземпляров (копий) СД.
  • Пункт назначения. Ссылается на справочник «Пункты назначения». Пункт получения СД. 
  • Владелец документа. Ссылается на справочник «Контрагенты». Контрагент - владелец СД. Если документ принадлежит нашей организации, то поле не отображается.
  • Основание. Тип значения: документ "Заказ на ТС" или "Маршрутный лист". Документ, к которому привязан СД.
  • Дата обработки (плановая). Тип значения: «Дата». Плановая дата завершения обработки (получения) СД. Фактическая же дата получения СД отличается от плановой и равна моменту установки статуса СД  "Обработка завершена".
    • Авто. Значение недоступно для редактирования и рассчитывается автоматически по данным табличной части "Виды документов" и реквизита "Способ расчета дней" справочника "Договоры контрагентов" (используется основной договор владельца документа) при проведении документов "Заказ на ТС" или "Маршрутный лист":
      • Плановая дата выполнения по Заказу на ТС. Значение даты обработки равно "Прибытие с" по Заказу плюс "Дней на обработку" календарных или рабочих. Если СД введен в Маршрутном листе, то при расчете плановой даты используется Заказ на ТС, введенный в Маршрутном листе. Если Заказов несколько, выбирается Заказ с самой ранней датой прибытия;
      • Фактическая дата выполнения по Маршрутному листу. Значение даты обработки равно "Дата выполнения фактическая" по Маршрутному листу плюс "Дней на обработку" календарных или рабочих. Если СД введен в Заказе на ТС, то при расчете плановой даты используется Маршрутный лист, введенный по этому Заказу на ТС. Если Маршрутных листов несколько, выбирается Маршрутный лист с самой ранней датой выполнения;
    • Вручную. Значение доступно для редактирования. Может указано как при создании нового СД так и позже в любое время.
  • Статус. Текущий статус СД.
  • Местоположение. Текущее местоположение СД.

Уведомления

Для СД может быть подключен механизм уведомлений об изменении статуса. Для этого нужно включить настройку программы "Использовать уведомления" и создать правило формирования уведомлений для СД по событию "Изменение статуса".
Создание уведомлений будет происходить при любом изменении статуса с помощью документов "Установка статуса сопроводительных документов" и "Пакет сопроводительных документов".
