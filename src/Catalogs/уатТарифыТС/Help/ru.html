Справочник "Тарифы на транспортные услуги"

Назначение

Справочник содержит информацию о прейскурантах и тарифах, согласно которым ведется расчет стоимости оказанных транспортных услуг.

Относится к подсистеме "Услуги".

Реквизиты

  • Прейскурант. Ссылается на справочник «Прейскуранты на транспортные услуги». Прейскурант, к которому относится этот тариф.
  • Услуга. Ссылается на справочник «Номенклатура». Номенклатура услуги, транспортные работы. Используется для подстановки в документы при расчете по тарифам. 
  • Тарифная сетка. Тип значения: Булево. Флаг позволяет указать, что тариф представляет собой тарифную сетку. Тарифная стека используется для ускорения расчета транспортных услуг для табличных тарифов. Строки таблицы - это возможные сочетания отборов области действия, а колонки - группировки. В ячейках содержатся значения тарифа. Например, при необходимости указания тарифов между различными городами в зависимости от объема груза в качестве областей действия (строки) указываются пары городов, в качестве группировок (колонки) - значения объема груза. В этом режиме ограничивается функционал ТЧ "Область действия": вид сравнения используется только "Равно", также можно указать список значений, и функционал ТЧ "Группировки": знак группировки возможен только "Меньше или равно".
  • Метод расчета. Перечисление "Методы расчета по тарифам". Может принимать значения: По параметру выработки, фиксированный, процентом от суммы. Определяет метод расчета стоимости услуг по тарифу:
    • По параметру выработки. Расчет производится по формуле: <Значение тарифа> * <Количество выработки>. Для сложных тарифов <Значение тарифа> определяется из таблицы "Группировки".
    • Фиксированный. Всегда берется <Значение тарифа>, количество услуги всегда равно 1.
    • Процентом от суммы. Расчет производится по формуле: <Базовая стоимость> * <Значение тарифа> / 100, где <Базовая стоимость> - стоимость "базовой" услуги, рассчитанная по базовому тарифу.
  • Параметр выработки. Ссылается на справочник «Параметры выработки». Параметр выработки, по которому отбирается выработка водителей.
  • Условие применения (фиксированного тарифа). Доступно только для фиксированных тарифов. Определяет применение фиксированного тарифа при нулевой выработке. "По условию" - тариф будет применен только при наличии выработки по параметру, указанному в тарифе. "Всегда" - тариф применяется независимо от наличия выработки.
  • Простой / Сложный "Сложным" считается тариф, размер которого определяется по совокупности условий, согласно таблице расчета (закладка "Группировки").
  • Скидка / Наценка. Доступен для тарифов с методом расчета "Процентом от суммы". Наценка увеличивает стоимость базовых услуг, скидка - уменьшает. Наценки используются во всех документах, имеющих табличные части "Доходы" и "Расходы", скидки только в тех документах, где в табличных частях есть колонки "Скидка" (доходы в документах Заказ на ТС, Потребность в перевозке).
  • Добавлять наценку отдельной строкой. Если флаг установлен, то в табличной части Доходы или Расходы будет добавлена новая строка с рассчитанной суммой наценки. Если флаг снят, то наценка отражается в строках с базовыми услугами в колонках "Процент скидки" и "Сумма скидки". При этом сумма скидки отображается положительным числом, сумма наценки - отрицательным.
  • Тип пункта. Тип значения: перечисление "Типы пунктов назначения". Значение будет использоваться в качестве отбора при расчете услуг по тарифам. Реквизит доступен только для параметра выработки "Количество точек по типу пункта".
  • Тип точки маршрута. Тип значения: перечисление "Типы точек маршрута". Значение будет использоваться в качестве отбора при расчете услуг по тарифам. Реквизит доступен только для параметра выработки "Количество точек маршрута".
  • Количество бесплатных единиц. Тип значения: «Число». При расчете стоимости по тарифу количество выработки уменьшается на это значение. Если количество выработки меньше или равно количеству бесплатных единиц, то количество считается равным нулю, и стоимость также равна нулю. Настройки минимального тарифа применяются после количества бесплатных единиц, даже если из-за бесплатных единиц количество стало равным 0.
  • Тариф. Тип значения: «Число». Размер тарифа в валюте управленческого учета, либо в процентах для метода расчета "Процентом от суммы". Используется только для "простого" тарифа.
  • Минимальный тариф
    • Минимальное количество. Тип значения: «Число». Минимальное количество по указанному параметру выработки, используемое при расчете стоимости услуг по тарифам. Если количество в документе меньше, чем минимальное количество по тарифу, то расчет выполняется по минимальному количеству.
    • Минимальная стоимость. Тип значения: «Число». Минимальная стоимость услуги по указанному параметру выработки. Если полученная в результате расчета стоимость услуги в документе меньше, чем минимальная стоимость по тарифу, расчет выполняется по минимальной стоимости.
    • Способ заполнения количества услуги в документе при использовании минимального тарифа:
      • "Количество равно фактическому". Количество услуги в документе не меняется, а цена услуги будет пересчитана по минимальной стоимости по тарифу, то есть: <Минимальная выработка> * <Значение тарифа> / <Количество выработки>. В этом случае цена услуги оказывается не равной значению тарифа.
      • "Количество равно минимальному". Количество и стоимость услуги в документе устанавливается равным минимальному количеству и стоимости, указанных в тарифе. В этом случае цена услуги оказывается равной значению тарифа.
      • "Количество равно 1". Количество услуги в документе устанавливается равным 1. Остальной расчет аналогичен предыдущему пункту.

Табличная часть «Таблица расчета»

Табличная часть предназначена для настройки "сложных" тарифов (определение "сложного" тарифа см. выше). Настройка заключается в указании интервалов действия того или иного значения тарифа. Например, в зависимости от расстояния нужно определить следующие значения тарифа: до 100 км - 200 руб, от 100 до 200 км - 250 руб, свыше 200 км - 300 руб. Тогда нужно указать группировку 1 - "Расстояние" и ввести 3 строки: < 100 - 200, >= 100 И < 200 - 250, >= 200 - 300.
Если в таблице есть несколько строк, удовлетворяющих условиям расчета, то будет выбран первый попавшийся тариф, начиная с первой строки. Поэтому рекомендуется использовать вид сравнения "<" или "<=" и нарастающие значения сравнения (или ">" или ">=" и убывающие значения сравнения), в противном случае может быть выбран неверный тариф.

  • Значение группировки 1-3. Тип значения: «Число». Значение группировки 1.
  • Знак группировки 1-3. Ссылается на перечисление «Знаки сравнения». Знак группировки №1-3.
  • Тариф. Тип значения: «Число». Размер тарифа в валюте упр. учета. Используется для "сложных" тарифов.

Табличная часть «Область действия»

Табличная часть предназначена для настройки фильтров действия данного тарифа. Настройка выполняется с помощью стандартного механизма отборов (аналогичный механизм используется при настройке отборов в отчетах). Каждая строка отбора будет работать как условие "И". То есть, например, если указаны значения Контрагент, Тягач и Груз, то тариф будет применяться в контексте перевозки указанного груза указанным тягачом для указанного контрагента. Если требуется указать несколько значений (например, несколько контрагентов или ТС), на которые будет действовать тариф, необходимо их добавить одной строчкой отбора с видом сравнения "В списке" и в значении указать нужный список значений.

Табличная часть «Базовые услуги»

Табличная часть предназначена для настройки базовых тарифов для тарифа с методом расчета "Процентом от суммы". Все тарифы, услуги в которых соответствуют услугам из этой табличной части, считаются базовыми для текущего.

Формы

  • Форма группы. Основная форма группы. Используется для заполнения обязательных реквизитов группы.
  • Форма списка. Основная форма списка. Используется для отображения списка элементов.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов.

Расчет по тарифам в документах

Расчет по тарифам на транспортные услуги выполняется в документах ТТД (табличная часть "Стоимость", кнопка "Рассчитать"), Заказ на ТС (табличные часть "Доходы" и "Расходы", кнопка "Рассчитать по тарифам"), Заказ перевозчику (табличная часть "Расходы", кнопка "Рассчитать по тарифам"), Потребность в перевозке (табличные часть "Предварительные доходы" и "Предварительные расходы", кнопка "Рассчитать по тарифам"), Маршрутный лист (табличные часть "Доходы" и "Расходы", кнопка "Рассчитать по тарифам"), Страховой сертификат (табличная часть "Расходы", кнопка "Рассчитать по тарифам").

На первом этапе запрашиваются действующие на дату документа прейскуранты и тарифы. Если выполняется расчет Доходов, то запрашиваются прейскуранты предприятия, если Расходов - то прейскуранты поставщиков, или если перевозка осуществляется собственным ТС, то показатели себестоимости. Далее по каждому параметру выработки (имеется ввиду рассчитанная выработка по документу) отдельно отбирается список тарифов и находится тариф, наиболее подходящий по области действия. Это означает, что область действия наиболее полно соответствует параметрам документа. Например, если один тариф имеет область действия "Маршрут = Сокольники-Лужники, Тип ТС = Грузовые средней вместимости", а другой "Маршрут = Сокольники-Лужники", то применится только первый. Разумеется вся область действия тарифа должна удовлетворять значениям соответствующих параметров документа. Если отберутся несколько тарифов с одинаковой степенью соответствия (степень - это количество соответствующих отборов области действия), то случайным образом система оставит только один. То есть в итоговой таблице услуг для каждого параметра выработки будет только одна рассчитанная по тарифу строка, за исключением случая прейскуранта предприятия с приоритетом тарифов "Все тарифы".
Тарифы запрашиваются сложные и простые, стандартные и вида "тарифная сетка" с равным приоритетом.
Тарифы с методом расчета "Процентом от суммы" рассчитываются после остальных тарифов. Тарифы "Процентом от суммы" выбираются только из прейскуранта базового тарифа. Для тарифов "Процентом от суммы" и "Фиксированный" всегда устанавливается количество = 1.
В документах Заказ на ТС, Заказ перевозчику, Потребность в перевозке, Маршрутный лист, Страховой сертификат также есть кнопки пересчета по тарифам "Пересчитать в текущей строке", "Пересчитать во всех строках"и т.п. Эти кнопки позволяют выполнить частичный расчет по тарифам по каждой текущей строке или по всем строкам табличной части:

   1) если тариф в строке заполнен, то он при расчете не изменяется. Количество в строке табличной части перезаполняется количеством выработки по соответствующему параметру выработки. Выполняется пересчет суммы;
   2) если тариф в строке не заполнен, то выполняется его поиск по указанной в строке услуге (и параметрам поставщика, если это табличная часть "Расходы" документа Маршрутный лист). Количество в строке табличной части перезаполняется количеством выработки по соответствующему параметру выработки. Выполняется пересчет суммы.

Особенности расчета по тарифам складского хранения

При расчете по параметру выработки "Складская обработка" и виду складской операции "Хранение" сумма хранения также зависит от числа дней хранения (с учетом нормы бесплатного хранения). Это справедливо как для тарифов с методом расчета "Фиксированной суммой" и условием применения по этому параметру выработки, так и для тарифов с методом расчета "По параметру выработки". Реквизит "количество бесплатных единиц" в данном случае не применяется.
При расчете по параметру выработки "Складская обработка" с видом складской операции, отличной от "Хранение", база тарифа рассчитывается с учетом количества бесплатных единиц.
