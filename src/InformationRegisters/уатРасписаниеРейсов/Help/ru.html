Регистр сведений "Расписание рейсов"

Назначение

Регистр предназначен для хранения расписание движения транспорта по рейсам. В зависимости от владельца рейса возможны 2 типа расписаний - по маршрутам и по пунктам назначения.

Измерения

  • Маршрут. Измерение используется, когда расписание задано для маршрутов.
  • Выход. Учет выходов может быть отключен. Измерение используется, когда расписание задано для маршрутов.
  • Рейс. Измерение используется, когда расписание задано для маршрутов.
  • Пункт отправления. Измерение используется, когда расписание задано для пунктов.
  • Пункт прибытия. Измерение используется, когда расписание задано для пунктов.
  • Грузоотправитель. Измерение используется, когда расписание задано для пунктов.
  • Грузополучатель. Измерение используется, когда расписание задано для пунктов.
  • УИД пункта. Текущий пункт маршрута, для которого действует строка расписания. Измерение используется, когда расписание задано для маршрутов.
  • Дата установки. Дата установки расписания (более свежие расписания считаются более приоритетными).
  • Действует с. Дата начала периода действия данного графика. Если не заполнено - действует без ограничения периода.
  • Действует по. Дата окончания периода действия данного графика. Если не заполнено - действует без ограничения периода.

Ресурсы

  • Дни недели (Пн,Вт, Ср...). 7 отдельных реквизитов на каждый день недели. Отправление рейса происходит по тем дням недели, по которым установлены флаги.
  • Дней до отправления. Заказы, отправляемые по этому расписанию, должны быть созданы заранее на это количество дней до даты отправления. Используется для автоматического заполнения дат отправления и прибытия в Заказе на ТС и Потребности в перевозке.
  • Дней в пути. Количество дней, выделенное на движение между пунктом отправления и прибытия. Используется для автоматического заполнения дат отправления и прибытия в Заказе на ТС. Только в версии КОРП реквизит используется в Потребности в перевозке. Ресурс используется, только когда расписание задано для пунктов. 
  • FTL. Флаг автоматически устанавливает признак FTL в Заказе на ТС или Потребности в перевозке, для которой используется расписание. Только в версии КОРП.

Относится к подсистеме "Билеты и выручка".
